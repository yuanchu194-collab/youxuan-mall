# 05-常见问题排查

## Nacos 8848 无法启动

现象：`docker compose` 启动 Nacos 失败，或服务无法注册到 `localhost:8848`。

原因：当前 Windows 机器的 8848 在 TCP 排除范围内。

处理：

```powershell
docker start youxuan-nacos-stage15
```

或新建临时 Nacos：

```powershell
docker run -d --name youxuan-nacos-stage19 -e MODE=standalone -e TZ=Asia/Shanghai -p 18848:8848 -p 19848:9848 -p 19849:9849 --network youxuan-mall-net nacos/nacos-server:v2.2.3
```

启动所有服务时追加：

```powershell
-Dspring-boot.run.arguments="--spring.cloud.nacos.discovery.server-addr=localhost:18848"
```

不要修改 `docker/docker-compose.yml` 来绕过这个本机端口问题，除非后续明确决定调整统一端口。

## MySQL 初始化 SQL 没有自动执行

现象：新增阶段的表不存在，例如 `home_banner`、`home_recommend`。

原因：MySQL Docker 数据卷已经初始化过，`docker/mysql/init/01-init.sql` 只在首次初始化数据目录时自动执行。

处理：

```powershell
docker exec -it youxuan-mysql mysql -uroot -proot youxuan_mall
```

进入后手动执行 README 或 `docker/mysql/init/01-init.sql` 中对应建表 SQL。

## Gateway 返回 401

现象：除注册、登录、`/test/ping` 外，请求返回未登录。

原因：缺少 `Authorization` 请求头，或 token 格式不正确。

处理：

```text
Authorization: Bearer <token>
```

先调用：

```http
POST /api/user/login
```

再把返回的 `data.token` 放入后续请求头。

## Gateway 返回 503 或找不到服务

现象：Gateway 能启动，但访问 `/api/product/**`、`/api/order/**` 等返回服务不可用。

排查：

1. 确认对应微服务已经启动。
2. 确认服务启动参数使用 `localhost:18848`。
3. 打开 `http://localhost:18848/nacos` 查看服务实例。
4. 查看 Gateway 日志中是否订阅到对应服务。

## MinIO 上传失败

现象：`POST /api/file/product/image` 失败。

排查：

1. 确认 MinIO 正常：

   ```powershell
   curl.exe http://localhost:9100/minio/health/live
   ```

2. 确认上传字段名是 `file`。
3. 确认文件类型是 `jpg`、`jpeg`、`png` 或 `webp`。
4. 确认文件大小不超过 5MB。

成功返回的 URL 应类似：

```text
http://localhost:9100/youxuan-mall/products/2026/06/xxx.png
```

## 商品详情缓存看不到命中

现象：接口能返回商品，但不确定 Redis 是否命中。

处理：

连续请求两次：

```http
GET /api/product/{id}
```

检查 Redis：

```powershell
docker exec youxuan-redis redis-cli EXISTS youxuan:product:detail:{productId}
```

返回 `1` 表示缓存存在。

## Elasticsearch 搜索不到商品

排查：

1. 确认 Elasticsearch 可访问：

   ```powershell
   curl.exe http://localhost:9200
   ```

2. 先执行全量导入：

   ```http
   POST /api/search/product/importAll
   ```

3. 修改商品后等待 RabbitMQ 消费几秒钟再搜索。

## RabbitMQ 同步不生效

排查：

1. 确认 RabbitMQ 运行：

   ```text
   http://localhost:15672
   guest / guest
   ```

2. 确认 `youxuan-product-service` 和 `youxuan-search-service` 都已启动。
3. 修改商品后等待 5 到 10 秒。
4. 查看 `stage19-youxuan-search-service.out.log` 中是否有消费日志或错误。

## 优惠券领取后我的优惠券暂时查不到

原因：领取成功后通过 RabbitMQ 异步写入 `user_coupon`。

处理：领取后等待几秒再查询：

```http
GET /api/coupon/my
```

## 发货接口返回 403

原因：发货接口只允许 `ADMIN` 角色调用。

处理：

1. 使用管理员账号登录获取 token。
2. 请求头使用管理员 token。

```http
POST /api/order/{id}/ship
Authorization: Bearer <admin-token>
```

## 超时订单没有立刻取消

原因：订单超时取消通过 RabbitMQ 死信队列处理，测试阶段延迟约 1 分钟。

处理：

1. 创建订单后不要支付。
2. 等待至少 75 秒。
3. 查询订单详情：

   ```http
   GET /api/order/{id}
   ```

4. 预期状态为 `4` 已取消。

## 首页聚合缓存没有命中

处理：

连续请求两次：

```http
GET /api/home/index
```

检查 Redis：

```powershell
docker exec youxuan-redis redis-cli EXISTS youxuan:home:index
```

返回 `1` 表示缓存存在。

新增、修改、删除 Banner，或新增、删除推荐位后，首页缓存会被删除，这是预期行为。
