# 优选商城未实现与待优化功能清单

## 1. 文档说明

本文档用于记录当前项目中：

- 尚未实现的功能；
- 前端已有入口但后端接口缺失的功能；
- 目前仅做提示、Mock、占位的功能；
- 已实现但仍需优化的功能；
- 后续可继续扩展的功能。

记录依据为当前仓库代码、前端路由、前端接口封装、后端 Controller、Docker 配置、README 和演示数据脚本。无法从代码确认的信息统一标注为“待确认”。

## 6. 页面层面的待优化项

### 6.1 首页

- 首页已调用 `homeApi.index()` 获取 Banner、分类、热门商品、推荐商品、新人优惠券。
- Banner 已有兜底资源，仍需确认线上演示数据中的图片风格是否统一。
- 商品图通过真实接口优先返回 `mainImage`，缺失时使用统一占位图。
- 推荐商品图是否全部为真实商品图取决于演示数据和 MinIO 图片是否已准备完成。

### 6.2 优惠券页面

- `/coupons` 已改为新版优惠券中心，包含账户侧边栏、Banner、Tab、优惠券卡片、空状态。
- `GET /api/coupon/page` 已限定为前台可领取券列表，仅返回已上线、已开始、未过期、库存大于 0、未逻辑删除的优惠券。
- 单张优惠券通过卡片最大宽度控制，避免横向拉满。
- 领取按钮调用真实 `POST /api/coupon/{id}/receive`，成功后刷新列表。
- 去使用按钮和查看可用商品跳转 `/products?couponId=xxx`，不伪造使用成功。
- 规则弹窗为前端展示规则说明；最终核销仍以后端订单创建为准。
- 分享、订阅提醒、高级筛选仍依赖后端接口或组件。

### 6.3 商品详情页

- 商品详情读取真实 `GET /api/product/{id}`。
- 加入购物车调用真实 `POST /api/cart/add`。
- 立即购买当前走“加入购物车后去购物车结算”的链路，不是独立立即购买订单。
- 规格选择、查看优惠、店铺入口使用 `showBackendTodo` 提示。
- 商品评价区域和评价接口尚未实现。

### 6.4 购物车

- 勾选商品、修改数量、删除商品、批量删除已选商品均接入真实购物车接口。
- 去结算会把已选商品带到结算页。
- 清空购物车、移入收藏、降价提醒、运费说明、购物车优惠金额计算、购物车优惠券入口仍未真实实现。
- 优惠金额最终应由订单确认和订单创建接口校验，不能只在前端扣减。

### 6.5 结算页

- 地址选择、默认地址、新增地址依赖真实地址接口。
- 订单确认调用 `POST /api/order/confirm`。
- 可用优惠券调用 `GET /api/coupon/available?amount=当前商品总额`，并兼容订单确认接口返回的 `availableCoupons`。
- 选择优惠券后会重新调用 `POST /api/order/confirm`，页面金额以新的后端确认结果展示。
- 提交订单调用 `POST /api/order`，可传递 `couponId`。
- 订单服务创建订单时会重新校验库存、地址、优惠券并执行优惠券核销。
- 第三方支付、余额支付、发票、配送方式、运费险仍为提示或展示项。

### 6.6 后台管理

- 商品管理：新增、编辑、上下架、删除、上传商品图片已接入真实接口。
- 订单管理：管理员分页、状态筛选、发货已接入真实接口。
- 优惠券管理：新增、编辑、上线、下线、逻辑删除、预热库存已接入真实接口，后台分页使用 `GET /api/coupon/admin/page`。
- Banner 管理：后端已有接口，前端后台页面未实现。
- 推荐位管理：后端已有接口，前端后台页面未实现。
- 文件上传：后台商品主图上传接入 MinIO 文件服务。

## 7. 后续开发优先级建议

### P0：必须补齐

- 保证 Docker 中间件、Nacos、后端服务、前端启动流程稳定。
- 首页、商品列表、商品详情、购物车、结算、订单主链路可跑通。
