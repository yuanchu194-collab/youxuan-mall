# 优选商城最终启动与演示文档

## 1. 项目说明

- 项目名称：优选商城
- 项目定位：Java 后端 + Vue 前端的商城项目
- 当前用途：学习、展示、面试演示、后续继续扩展
- 后端技术栈：Java 17、Spring Boot 3.2.6、Spring Cloud 2023.0.2、Spring Cloud Alibaba 2023.0.1.0、Spring Cloud Gateway、OpenFeign、MyBatis-Plus 3.5.7、MySQL、Redis、RabbitMQ、Nacos、Elasticsearch 7.17.18、MinIO、Knife4j、Maven
- 前端技术栈：Vue 3、Vite 6、TypeScript、Pinia、Vue Router、Axios、Element Plus、CSS
- 中间件：Docker、MySQL 8.0、Redis 7.2、Nacos 2.2.3、RabbitMQ 3-management、Elasticsearch 7.17.18、MinIO

## 3. Docker 容器说明

| 容器名 | 作用 | 端口 | 账号 | 密码 | 启动方式 |
|---|---|---|---|---|---|
| `youxuan-mysql` | MySQL 数据库 | `3307:3306` | `root` | `root` | `docker start youxuan-mysql` |
| `youxuan-redis` | Redis 缓存 | `6379:6379` | 无 | 无密码，compose 未配置 | `docker start youxuan-redis` |
| `youxuan-nacos` | 注册配置中心 | `8848:8848`、`9848:9848`、`9849:9849` | `nacos`（README 记录，compose 未显式设置） | `nacos`（README 记录，compose 未显式设置） | `docker start youxuan-nacos` |
| `youxuan-rabbitmq` | 消息队列 | `5672:5672`、`15672:15672` | `guest` | `guest` | `docker start youxuan-rabbitmq` |
| `youxuan-elasticsearch` | 搜索引擎 | `9200:9200` | 无，`xpack.security.enabled=false` | 无 | `docker start youxuan-elasticsearch` |
| `youxuan-minio` | 文件对象存储 | `9100:9000`、`9101:9001` | `minioadmin` | `minioadmin` | `docker start youxuan-minio` |
| `youxuan-minio-init` | MinIO bucket 初始化 | 无宿主机端口 | `minioadmin` | `minioadmin` | `docker compose -f docker/docker-compose.yml up minio-init` |

说明：

- Docker Compose 文件位置：`D:\code\youxuan-mall\docker\docker-compose.yml`。
- MinIO bucket：`youxuan-mall`，`minio-init` 会设置匿名下载。
- README 记录本机曾因 Windows TCP 保留端口导致 `8848` 不可用，本地联调可使用 `localhost:18848` 的 Nacos 临时方案。若你的机器 `8848` 可用，可直接使用 compose 中的 `youxuan-nacos`。
- 本次整理已从 `docker-compose.yml` 和服务配置确认容器名、端口、账号密码；尝试读取当前 Docker 运行态时被 Docker Desktop named pipe 权限拒绝，实际容器是否正在运行需在本机执行 `docker ps -a` 再确认。

## 4. 一键启动命令

### 4.1 启动 Docker 中间件

PowerShell 推荐：

```powershell
Set-Location D:\code\youxuan-mall
docker compose -f docker\docker-compose.yml up -d mysql redis rabbitmq elasticsearch minio
docker compose -f docker\docker-compose.yml up -d nacos
docker compose -f docker\docker-compose.yml up minio-init
docker ps --format "table {{.Names}}\t{{.Status}}\t{{.Ports}}"
```

如果容器已经创建，也可以逐个启动：

```powershell
docker start youxuan-mysql
docker start youxuan-redis
docker start youxuan-nacos
docker start youxuan-rabbitmq
docker start youxuan-elasticsearch
docker start youxuan-minio
docker ps --format "table {{.Names}}\t{{.Status}}\t{{.Ports}}"
```

如果本机 `8848` 不可用，按 README 中的联调记录，后端启动参数需要统一覆盖 Nacos 地址为：

```text
--spring.cloud.nacos.discovery.server-addr=localhost:18848
```

### 4.2 启动后端服务

#### IDEA 启动方式

1. 打开 `D:\code\youxuan-mall`。
2. 确认 Docker 中间件和 Nacos 已启动。
3. 依次启动：
   - `youxuan-user-service`
   - `youxuan-product-service`
   - `youxuan-search-service`
   - `youxuan-file-service`
   - `youxuan-coupon-service`
   - `youxuan-order-service`
   - `youxuan-cart-service`
   - `youxuan-home-service`
   - `youxuan-gateway`
4. 如果使用 `localhost:18848` 的 Nacos，需要给每个服务增加 VM 或 Program 参数覆盖 Nacos 地址。

#### Maven 命令启动方式

PowerShell 单服务启动示例：

```powershell
Set-Location D:\code\youxuan-mall
mvn -pl youxuan-user-service -am spring-boot:run
```

如果使用 `localhost:18848` 的 Nacos，启动命令示例：

```powershell
Set-Location D:\code\youxuan-mall
mvn -pl youxuan-user-service -am spring-boot:run -Dspring-boot.run.arguments="--spring.cloud.nacos.discovery.server-addr=localhost:18848"
```

各服务命令：

```powershell
Set-Location D:\code\youxuan-mall
mvn -pl youxuan-user-service -am spring-boot:run
mvn -pl youxuan-product-service -am spring-boot:run
mvn -pl youxuan-search-service -am spring-boot:run
mvn -pl youxuan-file-service -am spring-boot:run
mvn -pl youxuan-coupon-service -am spring-boot:run
mvn -pl youxuan-order-service -am spring-boot:run
mvn -pl youxuan-cart-service -am spring-boot:run
mvn -pl youxuan-home-service -am spring-boot:run
mvn -pl youxuan-gateway -am spring-boot:run
```

建议每个服务使用独立终端，便于停止和查看日志。

### 4.3 启动前端

PowerShell：

```powershell
Set-Location D:\code\youxuan-mall\youxuan-mall-web
npm install
npm run dev -- --host 127.0.0.1 --port 3000
```

CMD 兼容写法：

```cmd
cd /d D:\code\youxuan-mall\youxuan-mall-web
npm run dev -- --host 127.0.0.1 --port 3000
```

注意：不要在 `C:\Users\xxx` 目录直接运行 `npm run dev`，否则会报 `package.json` 找不到。

## 5. 一键关闭命令

### 5.1 关闭前端

在前端终端按：

```text
Ctrl + C
```

### 5.2 关闭后端

- IDEA 启动：点击 Stop。
- Maven 启动：在对应服务终端按 `Ctrl + C`。

### 5.3 停止 Docker 容器

只停止本项目容器：

```powershell
docker stop youxuan-mysql
docker stop youxuan-redis
docker stop youxuan-nacos
docker stop youxuan-rabbitmq
docker stop youxuan-elasticsearch
docker stop youxuan-minio
```

不要使用 `docker stop $(docker ps -q)`，该命令会误停其他项目容器。

## 9. 常见问题与解决办法

### 9.1 npm run dev 报 package.json 找不到

原因：

没有进入前端目录。

错误示例：

```text
在 C:\Users\xxx 目录执行 npm run dev
```

正确命令：

```powershell
Set-Location D:\code\youxuan-mall\youxuan-mall-web
npm run dev -- --host 127.0.0.1 --port 3000
```

CMD 兼容写法：

```cmd
cd /d D:\code\youxuan-mall\youxuan-mall-web
npm run dev -- --host 127.0.0.1 --port 3000
```

### 9.2 Docker 容器已经启动但服务无法访问

检查：

```powershell
docker ps
docker ps -a
netstat -ano | findstr :3307
netstat -ano | findstr :6379
netstat -ano | findstr :8848
netstat -ano | findstr :9000
netstat -ano | findstr :9100
netstat -ano | findstr :9101
```

重点确认：

- Docker Desktop 是否已启动；
- 端口是否冲突；
- Nacos 是否启动成功；
- 后端服务是否注册到 Nacos；
- Gateway 是否启动在 `9000`。

### 9.3 MySQL 连接失败

检查：

- `youxuan-mysql` 容器是否启动；
- 宿主机端口是否为 `3307`；
- 用户名密码是否为 `root/root`；
- 数据库 `youxuan_mall` 是否已初始化；
- 服务配置中的 JDBC URL 是否指向 `localhost:3307`。

### 9.4 前端接口请求失败

检查：

- 后端服务是否启动；
- Gateway 是否启动；
- Nacos 服务发现是否正常；
- Vite 代理和前端请求路径是否仍以 `/api/...` 访问；
- 浏览器是否携带登录 Token；
- 普通用户是否误访问 `/admin`。

### 9.5 优惠券无法使用

说明：

优惠券最终核销必须依赖后端订单结算接口。当前后端已有 `GET /api/coupon/available`、`POST /api/coupon/use` 和订单创建时的 `couponId` 校验链路，但前端不能单独伪造“使用成功”。演示时应通过领取优惠券、跳转商品列表、进入结算页、提交订单来验证最终结果。
