## 0. 当前目标

当前后端功能已经基本完成，前端还没有正式开始整体视觉优化。本阶段目标是：

1. 以 `docs/ui-reference` 中的页面参考图为视觉标准。
2. 以 `public/banners` 中的 4 张 Banner 图作为首页轮播素材。
3. 重构前端页面视觉，使项目更像真实商城，而不是普通后台模板。
4. 保持真实接口优先，不用假数据掩盖接口问题。
5. 后端没有实现的能力，可以先在前端保留入口、空状态、mock fallback，但必须集中管理，方便后续切真实接口。

## 2. 总体限制

Codex 执行时必须遵守：

1. 本阶段只做前端视觉优化和必要的前端交互整理。
2. 不进入阶段 20。
3. 不实现 Canal、Sentinel、Seata、秒杀。
4. 不重构后端主流程。
5. 不修改 `docker/docker-compose.yml`。
6. 不删除已有业务功能。
7. 不用静态假页面替换真实业务页面。
8. 真实接口优先，mock 数据只能作为 fallback。
9. mock 数据必须集中放在 `src/mock` 或 `src/constants`，不能散落在各个 `.vue` 文件里。
10. 每完成一个小阶段，都要执行 `npm run build`。
11. 如果没有明确要求，不要修改 Java 后端代码。
12. 如果确实需要改接口字段，必须先说明原因，并做最小修改。

## 3. 设计规范

### 3.1 主视觉方向

整体参考这些关键词：

```text
浅色、现代、干净、卡片式、圆角、柔和阴影、绿色生鲜商城风格、适合截图展示
```

### 3.2 色彩建议

建议创建统一主题变量，例如：

```css
:root {
  --yx-primary: #16a34a;
  --yx-primary-dark: #15803d;
  --yx-primary-light: #dcfce7;

  --yx-bg: #f6fbf5;
  --yx-bg-soft: #f0f9ef;
  --yx-card: #ffffff;

  --yx-text: #172033;
  --yx-text-soft: #4b5563;
  --yx-muted: #8a9a8f;

  --yx-orange: #ff7a1a;
  --yx-red: #ef4444;
  --yx-yellow: #f59e0b;
  --yx-blue: #3b82f6;

  --yx-border: #e5eee4;

  --yx-radius-sm: 8px;
  --yx-radius-md: 12px;
  --yx-radius-lg: 20px;
  --yx-radius-xl: 28px;

  --yx-shadow-card: 0 12px 32px rgba(15, 122, 53, 0.08);
  --yx-shadow-soft: 0 8px 24px rgba(31, 41, 55, 0.06);
}
```

### 3.3 统一组件风格

全站统一：

1. 页面背景：浅绿色或浅灰白。
2. 卡片：白色背景、圆角、柔和阴影。
3. 主按钮：绿色。
4. 促销按钮：橙色。
5. 价格：红橙色突出。
6. 商品图片：统一比例，不拉伸变形。
7. 表格：浅边框、宽松行高、状态标签用彩色 pill。
8. 空状态：不要空白，显示图标和提示。
9. 加载中：保留 loading 状态。
10. 错误状态：Element Plus Message 或页面提示。

## 5. mock 开关设计

在前端根目录创建或更新：

```text
.env.development
```

内容示例：

```env
VITE_API_BASE_URL=http://localhost:9000
VITE_USE_MOCK=false
```

规则：

```text
VITE_USE_MOCK=false：只使用真实后端接口
VITE_USE_MOCK=true：允许后端接口失败时使用 mock fallback
```

注意：

1. 不能默认全页面假数据。
2. mock 只用于后端未完成或接口暂时不可用时兜底。
3. 页面必须优先请求真实接口。
